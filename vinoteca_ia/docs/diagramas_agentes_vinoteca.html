01
Agente Enrutador

Primera puerta del sistema. Clasifica cada mensaje entrante en milisegundos y deriva al agente especializado correcto. Opera siempre a temperatura 0.0 — sin creatividad, máxima precisión.

Proceso interno
Guardrail / error
Decisión
→ Ag. Sumiller
→ Ag. Inventario
→ Ag. Pedidos
→ Ag. Eventos
02
Agente Sumiller

El corazón del sistema. Opera con ReAct: razona, actúa, observa, y recalibra la recomendación en cada paso. Temperatura 0.7 para generar recomendaciones ricas; 0.0 cuando toca datos duros.

Núcleo cognitivo (LLM)
Tool call RAG
Tool call SQL
Memoria
Error / Fallback
03
Agente de Pedidos

Maneja la transacción completa con Two-Phase Commit obligatorio. Todo a temperatura 0.0 — ninguna creatividad cuando hay dinero y stock de por medio. Idempotency key en cada mutación.

Fase 1 · Preparación (sin mutación)
Fase 2 · Ejecución (mutación protegida)
Operación que muta datos
Log inmutable
04
Agente de Inventario

Solo SQL. Nunca vectores para precios o stock. Responde consultas exactas: precio de un vino, disponibilidad por zona, comparativa de cosechas. Temperatura 0.0 siempre.

Consulta SQL (determinista)
Regla crítica
Validación semántica
Respuesta formateada
05
Agente Knowledge Pipeline

El agente que mantiene actualizado al sumiller. Corre en background de forma continua. Tres triggers distintos: scraping programado, captura del sumiller, y nuevo vino en catálogo.

Trigger A (scraping)
Trigger B (sumiller)
Trigger C (nuevo vino)
Indexación RAG
Clasificación capas
06
Agente de Soporte

Maneja reclamos, preguntas fuera del flujo estándar y situaciones emocionales. Su función crítica es saber cuándo escalar a humano sin dilatar el problema.

Consulta SQL (estado pedido)
RAG (FAQ / políticas)
Resolución autónoma
Escalada a humano
07
Agente LLM-as-a-Judge

Auditor nocturno completamente asíncrono. Evalúa todas las conversaciones del día con temperatura 0.0, modelo de familia diferente al agente auditado y sin acceso al razonamiento interno. Solo entradas y salidas.

Trigger asíncrono (nocturno)
Rúbrica de evaluación
Aprobado → Golden Dataset
Fallo → 3 acciones correctivas
